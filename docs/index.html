ASNUX v1.0

Moteur audio basse latence pour Linux

ASNUX est un moteur audio basse latence conçu pour remplacer ASIO4ALL sous Linux. Il fournit un driver noyau ALSA virtuel, un daemon systemd temps réel et une interface graphique eframe/egui écrite en Rust. Compatible avec n'importe quelle DAW supportant ALSA (REAPER, Ardour, Bitwig, LMMS...) et intégré à PulseAudio/PipeWire pour une utilisation transparente.

Installation en une commande

Détection automatique de votre distribution. Fonctionne sur Debian, Ubuntu, Fedora, Arch, openSUSE, Slackware et bien d'autres.

Vous pouvez aussi télécharger le script et l'inspecter avant exécution.

Fonctionnalités

Basse latence

Buffer configurable de 16 à 8192 échantillons. Priorité temps réel pour les flux audio.

Driver noyau ALSA

Module kernel virtuel pour une intégration système profonde. Multi-canaux jusqu'à 8 (7.1).

Interface graphique

GUI moderne pour régler buffer, sample rate, canaux et définir ASNUX comme moteur par défaut.

Multi-format

20 formats de packages : DEB, RPM, Arch, Gentoo, Alpine, Nix, Slackware, Void, Solus, AppImage, Flatpak, Snap, Docker, Homebrew...

Rates pro

Spécifications techniques

Système d'exploitation

Linux kernel 5.10+. Toutes distributions : Debian, Ubuntu, Fedora, Arch, openSUSE, Gentoo, Alpine, NixOS, Void, Slackware, Solus.

Architecture

x86_64 (AMD64). Support ARM64 (Raspberry Pi 5) en bêta. Module noyau en C, daemon en Rust, GUI en Rust/eframe.

Mémoire

~8 Mo RAM pour le daemon. ~30 Mo pour la GUI. Empreinte minimale adaptée aux systèmes embarqués et stations audio.

Latence

Buffer configurable de 16 à 8192 échantillons. Latence minimale théorique : 0.33 ms à 48 kHz/16 samples. Recommandé : 128-256 samples (2.7-5.3 ms).

Canaux audio

Jusqu'à 8 canaux (7.1 surround). Configurable par utilisateur. Mixage stéréo automatique si source multicanale.

Formats supportés

S16_LE, S24_LE, S32_LE, Float 32-bit. Taux d'échantillonnage : 8 kHz à 192 kHz. Compatible tout matériel audio reconnu par ALSA.

Configuration

Le module noyau ASNUX se charge avec les paramètres par défaut :

# Paramètres modulables (via /sys/module/asnux/parameters/)
buffer_size=256     # Taille du buffer (16-8192)
sample_rate=48000   # Taux d'échantillonnage (8000-192000)
channels=2          # Nombre de canaux (1-8)
periods=4           # Nombre de périodes (2-1024)

FAQ

ASNUX remplace-t-il complètement PulseAudio ?

Non. ASNUX est un moteur audio basse latence conçu pour les DAW et stations audio. Il coexiste avec PulseAudio/PipeWire. Vous pouvez basculer entre les deux via le panneau de configuration.

Quelle est la différence avec JACK ?

JACK est un serveur de routage audio. ASNUX est un driver noyau virtuel. Les deux sont complémentaires : ASNUX améliore la couche ALSA, JACK gère le routage entre applications. Utilisez les deux ensemble pour un setup professionnel.

Mes plugins VST Windows fonctionnent-ils ?

Oui, via Wine. ASNUX fournit des latences suffisamment basses pour le monitoring temps réel avec des plugins VST Windows émulés. Utilisez yabridge pour le pontage automatique.

Comment désinstaller ASNUX proprement ?

Selon votre format d'installation : sudo dpkg -r asnux (DEB), sudo rpm -e asnux (RPM), sudo pacman -R asnux (Arch). Le module noyau est automatiquement retiré.

ASNUX est-il compatible Secure Boot ?

Le module noyau nécessite d'être signé pour Secure Boot. Un guide de signature est inclus dans la documentation. Alternative : désactiver temporairement Secure Boot dans le BIOS.

Puis-je utiliser ASNUX avec un DAC USB ?

Oui. ASNUX fonctionne avec tout périphérique audio reconnu par ALSA, y compris les DAC USB, les interfaces audio FireWire, et les cartes son PCI/PCIe.

Comment savoir si ASNUX est bien actif ?

Vérifiez en quelques commandes :

# Récupérer le numéro de carte ASNUX
CARD=$(cat /proc/asound/cards | grep -i asnux | head -1 | awk '{print $1}')

lsmod | grep asnux                      # Module chargé ?
cat /proc/asound/cards | grep -i asnux   # Carte ALSA présente ?
aplay -l | grep -i asnux                  # Périphérique playback visible ?
cat /sys/module/asnux/parameters/buffer_size  # Paramètres accessibles ?
amixer -c $CARD scontrols | grep ASNUX   # Contrôles mixer présents ?
sudo dmesg | grep -i asnux               # Chargement sans erreur ?
ls /dev/snd/controlC$CARD                # Device node existant ?

Si toutes ces commandes retournent un résultat, ASNUX est opérationnel. Vous pouvez aussi tester le playback avec speaker-test -D hw:0,0 -c 2 -t sine -f 440.

Erreur "Invalid module format" au chargement du module ?

Cette erreur survient quand le module noyau asnux.ko a été compilé pour une version différente du noyau. Vérifiez avec modinfo asnux.ko | grep vermagic : le vermagic doit correspondre exactement à uname -r. Si ce n'est pas le cas, recompilez depuis les sources :

git clone https://github.com/devfrp/asnux.git
cd asnux/kernel
make clean
make
sudo cp asnux.ko /lib/modules/$(uname -r)/kernel/drivers/sound/
sudo depmod -a
sudo modprobe asnux

Cause typique : mise à jour du noyau sans recompilation du module, ou module pré-compilé pour un noyau différent (ex. Azure vs noyau local).
